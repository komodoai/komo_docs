---
title: run
---
Run any bash command on the backend and resource of your choice.

### Usage

`komo run [OPTIONS] [ARGS]...`


### Options

| Option                              	| Description                                                                                                                                                       	| Type 	|
|-------------------------------------	|-------------------------------------------------------------------------------------------------------------------------------------------------------------------	|------	|
| &#x2011;b, &#x2011;&#x2011;backend  	| Name of the backend to use. Defaults to the default backend specified in your project config.                                                                  	| Text 	|
| &#x2011;r, &#x2011;&#x2011;resource 	| Name of the resource to use. Defaults to the default resource specified in your project config.                                                                	| Text 	|
| &#x2011;d, &#x2011;&#x2011;detach   	| Exit after the job is started instead of following the logs                                                                                                       	|      	|
| &#x2011;c, &#x2011;&#x2011;config   	| Override any parameters in the section of your project config corresponding to the specified backend.<br />Values must be provided as a comma-separated list of key=value pairs. 	| Text 	|

### Examples

Run a simple hello world job, using your default backend and resource
```bash
komo run echo "Hello World"
```

Run the same job on your `aws` backend using resource `g4`
<Note>This assumes you have created a backend named `aws` and a resource named `g4` for that backend</Note>
```bash
komo run -b aws -r g4 echo "Hello World"
```

Run the same job, this time overriding the `workdir` specified in your
project config for backend `aws`
```bash
komo run -b aws -r g4 -c workdir=/app
```

Run a job and exit as soon as the job has started
```bash
komo run -d echo "Hello World"
```

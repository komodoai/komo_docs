---
title: add-aws-instance
---

Add an instance type to your AWS compute infrastructure that was created with `komo compute create -t aws`.

### Usage

`komo compute add-aws-instance [OPTIONS]`

### Options

| Option                              	    | Description                                                       	| Type 	|
|-----------------------------------------  |-------------------------------------------------------------------	|------	|
| &#x2011;n, &#x2011;&#x2011;name 	| The name/alias you want to give to this instance type. Required.                       | Text |
| &#x2011;t, &#x2011;&#x2011;instance-type  	            | The type of instance to add. Required.| Text  |
| &#x2011;h, &#x2011;&#x2011;max-vcpus  	            | The maximum number of vCPUs that can be in use for this instance type at any given time. Defaults to 10000. | Int  |
| &#x2011;v, &#x2011;&#x2011;volume-size  	            |  | Int size of the EBS volume, in GB, that will be attached to instances of this type upon launch. Defaults to 100. |
| &#x2011;m, &#x2011;&#x2011;manual-confirm | Manually confirm terraform plan.                                      |      	|


### Examples

Set up a p4d.24xlarge instance type.
```bash
komo compute add-aws-instance -n p4-24xl -t p4d.24xlarge -v 100
```

Create a Komodo machine with whatever resources you need.

### Usage

`komo machine create [OPTIONS] NAME`

### Options

| Option                              	| Description                                                                                                                                                       	| Type 	|
|-------------------------------------	|-------------------------------------------------------------------------------------------------------------------------------------------------------------------	|------	|
| &#x2011;c, &#x2011;&#x2011;cloud  	| Name of the cloud to use. If not provided, Komodo will select a cloud for you based on GPU prices and availability.                                                                  	| Text 	|
| &#x2011;g, &#x2011;&#x2011;gpus 	| Which gpus to use, in the format \<GPU type\>:\<Num GPUs\> (eg. A100:4)                                                                	| Text 	|
| &#x2011;d, &#x2011;&#x2011;detach   	| Do not wait for the machine to start up.            


### Examples

Create a machine named `dev` using the default cloud/resources provided in your project config file.
```bash
komo machine create dev
```

Create a machine named `dev` on AWS with a single T4 GPU

```bash
komo machine create --cloud aws --gpus T4:1 dev
```

Create a machine named `dev` using the default cloud/resources provided in your project config file, but don't wait for it to start up.
```bash
komo machine create -d dev
```
